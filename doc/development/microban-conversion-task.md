# Microban → One Step Dungeon 変換タスク

## 概要

Microbanの既存パズルをこのゲーム（One Step Dungeon）の形式に変換して、
難易度の高いステージを追加する。

行長は揃える必要はない（既存の「最後の一押し」「大迷宮」等も行長バラバラで動いている）。

---

## 変換ルール（シンプル）

| Microban記号 | このゲームの記号 | 意味 |
|---|---|---|
| `#` | `#` | 壁 |
| ` ` (スペース) | ` ` | 床 |
| `@` | `P` | プレイヤー |
| `$` | `B` | 石（ブロック） |
| `.` | `T` または `U` | スイッチ（人間が決める） |
| `*` | Bと重なる | 石がゴール上にある状態 |
| `+` | Pと重なる | プレイヤーがゴール上にある状態 |

**XゲートとGゴールはMicrobanにない概念なので、人間が追加位置を決める。**

---

## 変換対象：Microban #91

### 元のマップ（そのまま）

```
###########
# .## #
# $$@..$$ #
# ##. #
###########
```

### 変換後（$→B, @→P, .→T, Xは未配置）

```javascript
{
    name: "Microban#91変換（X/G未配置）",
    size: 11,
    map: [
        "###########",
        "# T## #",      // .→T（緑SW）
        "# BBP TT BB #", // $→B、@→P、.→T
        "# ##T #",      // .→T（緑SW）
        "###########",
        // ↓ この行以降にX（緑ゲート）とG（ゴール）を追加してください
    ],
    enemies: [],
},
```

**注意：** `# BBP TT BB #` の行は元の `# $$@..$$ #` と同じ文字数になるように確認すること。

---

## ゴール位置（元の.の座標）

size=11のグリッド上での座標：

```
###########   row0
# .## #       row1  → .at (2,1)
# $$@..$$ #   row2  → .at (5,2), .at (6,2)
# ##. #       row3  → .at (4,3)
###########   row4
```

ゴール.が4箇所ある → T（緑SW）またはU（青SW）に変換する。
どれをTにしてどれをUにするかは設計者が決める。

---

## 次のステップ（人間にお願いしたいこと）

1. T（緑SW）とU（青SW）の配置を決める（4つの.をどう分けるか）
2. X（緑ゲート）とY（青ゲート）をどこに追加するか決める
3. K（鍵）・D（扉）・G（ゴール）の配置を決める
4. ソルバーで検証する：`node work/stage-solver/solver.mjs`
