# コミットを勝手に実行しない

## ルール

**ユーザーから明示的に「コミットしてください」と指示されない限り、
`git commit` を実行してはならない。**

## 具体的な禁止行為

- タスクの実装が完了した後に自動的にコミットする
- 「実装完了」と判断して勝手にコミットする
- コミットメッセージを作成してコミットを実行する

## 許可される行為

- `git add`（ステージング）はユーザーの指示があれば可
- `git status` / `git diff` などの参照系コマンドは自由に実行可
- コミットが必要かどうかをユーザーに確認する

## 理由

コミットはプロジェクトの履歴に永続的に残る操作であり、
いつ・どのタイミングで・どのメッセージでコミットするかは
ユーザーが判断すべき事項である。
AI が勝手にコミットすると、意図しない履歴が作成される可能性がある。

## 違反した場合の対処

`git reset HEAD~1` でコミットを取り消し、未コミット状態に戻すこと。
